# Context Engine

> Building complete investment context before evaluating evidence.

---

## Purpose

The Context Engine is responsible for transforming structured knowledge into a relevant investment context.

Rather than exposing every available piece of information, the Context Engine selects only the knowledge required for a specific investment analysis.

It answers the question:

"What information is actually relevant before reasoning begins?"

---

## Mission

Build complete, relevant, and focused context packages that enable accurate investment reasoning.

---

## Philosophy

Information without context creates noise.

Context creates understanding.

Every investment decision must begin with sufficient context.

The Context Engine never performs reasoning.

It prepares context.

---

## Primary Objective

Generate a standardized Context Package containing only the information required for downstream Engines.

---

## Responsibilities

The Context Engine is responsible for:

- Selecting relevant knowledge
- Building investment context
- Filtering irrelevant information
- Connecting related knowledge
- Organizing context for reasoning
- Preparing Context Packages

## Success Criteria

The Engine succeeds when:

- Relevant information is selected.
- Irrelevant information is removed.
- Context remains complete.
- Context is traceable.
- Context is reproducible.

---

## Failure Conditions

The Engine must reject processing if:

- Knowledge Package is incomplete.
- Required context is missing.
- Context relationships cannot be established.

---
